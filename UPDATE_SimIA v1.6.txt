SimIA v1.6.1 — Global Branding Loader

Problem fixed
- Brand/logo configured in Admin Setup appears only on admin.html

Cause
- Other pages were not loading platform branding from /api/config

Solution
- Add a generic loader: /assets/branding.js
- Include it on every page that has a topbar brand block

Files included
- assets/branding.js
- PATCH_SNIPPETS.txt

How to install
1) Copy assets/branding.js into your repo
2) In every page that should show the configured logo/brand/subtitle, add before </body>:
   <script src="/assets/branding.js"></script>

Recommended pages
- admin.html
- access.html
- player.html
- observer.html
- instructor.html
- outputs_admin.html
- scenario_timeline.html
- index.html

Important
- The page must contain a header brand block with classes:
  .brand
  .logo
  .brand-title
  .brand-sub

Result
- The configured brand/logo/theme subtitle will be applied across pages automatically
